## JWT Auth Classification

- Claims-first by default:
  Routes using [`lib/auth/api-session.ts`](/Users/callumsoutar/Developing/FlightDesk/flight-desk/lib/auth/api-session.ts) now preserve the Supabase `getClaims()` fast path unless they explicitly opt into `requireUser: true`.
- Should explicitly keep `requireUser: true`:
  High-sensitivity mutation/admin handlers and callback-style routes that already ask for authoritative role or tenant checks, such as invite management, booking mutations, Xero export/connect actions, settings mutations, and similar admin workflows.
- Still direct but lower priority:
  Some direct `getAuthSession(...)` users only check presence of `user` and do not participate in the role or tenant trust-boundary issue directly, for example routes under aircraft components/types/charge-rates, observations, maintenance visits, and similar utility handlers.
